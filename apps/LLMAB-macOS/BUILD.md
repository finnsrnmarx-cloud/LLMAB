# Building ω (LLMAB-macOS)

## V1 path: open in Xcode

```sh
open Package.swift     # from repo root
```

Xcode picks up the SwiftPM workspace and exposes `LLMABApp` as a runnable
executable scheme. Set a Development Team and ⌘R — the app launches against
whatever local runtimes (Ollama, MLX, llama.cpp) are installed.

## CI path: headless compile

```sh
swift build --target LLMABApp
```

This builds the app's source files but does not produce a signed `.app`
bundle — SwiftPM executable targets are bare Mach-O binaries. CI runs this
as a regression gate.

## Packaging (chunk 16)

Chunk 16 lands:

1. A proper `.xcodeproj` (generated by XcodeGen from a checked-in
   `project.yml`).
2. Asset catalogs for the ω app icon at every required size.
3. A `scripts/package.sh` that builds → signs (Developer ID) → notarises →
   staples → wraps in a DMG.

Until then, the app scheme inherits the Info.plist / entitlements in this
directory when opened in Xcode.

## Runtime expectations

- **Ollama** is the only v1 runtime. Install with:

  ```sh
  brew install --cask ollama
  ollama pull gemma-4:e4b
  ```

- **MLX** and **llama.cpp** adapters arrive in chunk 14. Once landed, ω
  auto-detects them and surfaces their models in Settings.
